# Research Journal — Open VLM Representation Analysis

## Project Motivation

The goal of this repository is to understand how open vision-language models encode and align image and text representations.

This project follows from a previous Transformer Representation Analysis project, where text-only transformer embeddings were studied using:

* PCA
* UMAP
* cosine similarity
* class separation
* layer-wise analysis
* CKA
* attention analysis

The next natural step is to move from:

```text
text → embedding
```

to:

```text
image → embedding
text  → embedding
        ↓
shared representation space
```

This repository focuses on CLIP because CLIP provides the cleanest starting point for studying multimodal alignment. The goal is not to train a large vision-language model. The goal is to understand how pretrained multimodal embeddings behave.

This project is part of the broader preparation roadmap:

```text
Transformer Representation Analysis
        ↓
Open VLM / CLIP Representation Analysis
        ↓
Astronomy Data Analysis
        ↓
Reasoning Audit
```

---

## Core Research Question

The central question of this repository is:

```text
How do open vision-language models align image and text representations?
```

More specifically:

* Can CLIP encode images and captions into the same embedding dimension?
* Can an image retrieve its correct or semantically related caption?
* Are matching image-text pairs more similar than non-matching pairs?
* Do image embeddings and text embeddings form meaningful semantic clusters?
* Is there a visible modality gap between image and text embeddings?
* Which semantic classes are easiest or hardest for CLIP to separate?

The current project studies these questions using a small controlled dataset containing:

* galaxy
* dog
* cat
* car

## First CLIP Smoke Test

The first actual model experiment was a minimal image-text embedding inspection.

Input:

```text
image: data/samples/sample_image.jpg
text:  A galaxy with bright stars.
```

Result:

```text
Text embedding shape:  (1, 512)
Image embedding shape: (1, 512)
Cosine similarity:     0.1703
```

Interpretation:

CLIP maps both image and text inputs into the same 512-dimensional embedding space.

This was the Stage 2 equivalent of the first hidden-state shape inspection in the Transformer Representation project.

The first important milestone was therefore:

```text
image → CLIP image embedding → (1, 512)
text  → CLIP text embedding  → (1, 512)
```

## Image-to-Text Retrieval Analysis

The next experiment asked:

```text
Given an image embedding, can CLIP retrieve the correct caption?
```

Implemented:

```text
src/analysis/retrieval.py
scripts/run_retrieval_analysis.py
```

The analysis computes an image-to-text similarity matrix and predicts the most similar caption for each image.

### Initial 4-Sample Result

For the 4-sample dataset, CLIP retrieved:

```text
galaxy → galaxy caption
dog    → dog caption
cat    → dog caption
car    → car caption
```

Top-1 accuracy:

```text
3 / 4 = 75%
```

The cat image was incorrectly matched to the dog caption, but the similarity scores were close.

This suggested that cat/dog distinction was harder than galaxy/dog or galaxy/car distinction.

---

## Retrieval Analysis on 20 Samples

After expanding to 20 image-caption pairs, exact-caption retrieval looked worse because many predictions were marked `False`.

However, inspection showed that most of these were semantically correct at the class level.

Example:

```text
query image:
galaxy_1.jpg

correct caption:
A spiral galaxy with bright stars.

predicted caption:
An astronomical image of a galaxy.
```

This is not actually semantically wrong.

It means CLIP retrieved another galaxy caption.

### Important Insight

Exact-caption retrieval is too strict once there are multiple examples per class.

CLIP performs semantic retrieval, not file-name matching.

Therefore:

```text
image_i → caption_i
```

is not always the right success criterion.

A more meaningful metric is:

```text
image label == predicted caption label
```

For the 20-sample dataset, the retrieval results strongly suggested high class-level retrieval accuracy, especially for galaxy, dog, cat, and car class groups.

## Image-Text Similarity Matrix

A full image-text similarity matrix was computed using:

```text
scripts/run_similarity_analysis.py
```

Saved outputs:

```text
outputs/clip/analysis/similarity/
├── image_text_similarity_matrix.npy
└── image_text_similarity_matrix.csv
```

The matrix revealed clear structure.

### Galaxy Rows

Galaxy images had high similarity to galaxy captions:

```text
~0.28 to 0.34
```

and substantially lower similarity to dog, cat, and car captions.

### Dog Rows

Dog images had highest similarity to dog captions:

```text
~0.23 to 0.30
```

with lower similarity to other classes.

### Cat Rows

Cat images had high similarity to cat captions:

```text
~0.24 to 0.31
```

but also relatively high similarity to dog captions.

This indicated partial cat/dog overlap.

### Car Rows

Car images had highest similarity to car captions:

```text
~0.20 to 0.26
```

with lower similarity to most other classes.

### Interpretation

The similarity matrix shows that CLIP has strong class-level semantic organization.

It does not merely memorize exact pairs; it groups semantically related captions and images.

## Cross-Modal Alignment Analysis

Cross-modal alignment analysis was implemented in:

```text
src/analysis/cross_modal.py
scripts/run_cross_modal_analysis.py
```

This computes:

```text
matching_mean
non_matching_mean
alignment_gap
```

where:

```text
alignment_gap = matching_mean - non_matching_mean
```

### 4-Sample Result

```text
matching_mean     = 0.2342
non_matching_mean = 0.1400
alignment_gap     = 0.0943
```

Interpretation:

Matching image-text pairs were more similar than non-matching pairs.

### 20-Sample Result

```text
matching_mean     = 0.2712
non_matching_mean = 0.1856
alignment_gap     = 0.0856
```

At first, the smaller gap seemed worse. However, this was due to the metric becoming less clean after adding multiple samples per class.

With 20 samples, many technically non-matching pairs are still semantically related:

```text
galaxy_1 image ↔ galaxy_4 caption
dog_2 image    ↔ dog_5 caption
cat_3 image    ↔ cat_1 caption
```

These increase the non-matching mean.

### Interpretation

The cross-modal alignment gap remains positive, confirming that matching image-text pairs are closer than unrelated pairs.

However, once multiple examples per class are introduced, class-aware metrics become more informative than exact pair metrics.

## PCA and UMAP Visualizations

Visual analysis was implemented using:

```text
src/analysis/pca.py
src/analysis/umap.py
src/visualization/scatter.py
scripts/run_visual_analysis.py
```

Generated outputs:

```text
outputs/clip/visualizations/
├── image_pca.png
├── image_umap.png
├── text_pca.png
└── text_umap.png
```

### 4-Sample Observation

With only 4 samples, UMAP was not meaningful.

UMAP warned that:

```text
n_neighbors is larger than the dataset size
```

This was expected because UMAP requires more samples to build a meaningful neighborhood graph.

### 20-Sample Observation

After expanding to 20 samples, PCA and UMAP became much more interpretable.

### Image PCA

Image PCA explained variance:

```text
PC1 = 0.2798
PC2 = 0.2185
Total = 0.4982
```

Observation:

* Galaxy images formed a clearly distinct cluster.
* Car images formed another distinct region.
* Cat and dog images were closer and partially overlapping.

### Text PCA

Text PCA explained variance:

```text
PC1 = 0.2489
PC2 = 0.1745
Total = 0.4234
```

Observation:

* Text captions formed clear semantic clusters.
* Galaxy captions separated strongly from everyday-object captions.
* Dog, cat, and car captions were also mostly organized into distinct regions.

### Interpretation

Both image and text embedding spaces show meaningful low-dimensional semantic structure.

The image and text embedding spaces do not collapse into one dominant direction; information is distributed across multiple dimensions.

## Joint Image-Text PCA

Joint PCA was implemented using:

```text
scripts/run_joint_visual_analysis.py
src/visualization/joint_scatter.py
```

This combines image and text embeddings into the same PCA projection.

The joint plot connects matching image-text pairs with dashed lines.

### Joint PCA Explained Variance

```text
PC1 = 0.4063
PC2 = 0.1197
Total = 0.5260
```

### Observation

The joint PCA plot showed a clear modality gap:

```text
image embeddings → one side
text embeddings  → other side
```

The first principal component was largely capturing:

```text
image vs text
```

rather than only semantic class.

### Important Interpretation

CLIP does not make image and text embeddings identical in distribution.

Instead, the evidence suggests:

```text
CLIP embedding space = shared semantic structure + modality offset
```

This is an important multimodal representation finding.

The embeddings share enough semantic structure for retrieval and alignment, but still preserve a modality-dependent offset in low-dimensional projection.

## Class Separation Analysis

Class separation analysis was implemented in:

```text
src/analysis/class_separation.py
scripts/run_class_separation_analysis.py
```

This mirrors the class-separation analysis used in the Transformer Representation project.

The metric is:

```text
separation_score = within_similarity - between_similarity
```

where:

```text
within_similarity  = average image-text similarity within the same class
between_similarity = average image-text similarity across different classes
```

### Results

| Label  | Within Similarity | Between Similarity | Separation Score |
| ------ | ----------------: | -----------------: | ---------------: |
| galaxy |            0.3038 |             0.1779 |           0.1260 |
| dog    |            0.2515 |             0.1663 |           0.0852 |
| car    |            0.2229 |             0.1407 |           0.0822 |
| cat    |            0.2595 |             0.1787 |           0.0808 |

### Interpretation

Galaxy was the best-separated class.

Cat was the weakest-separated class.

This matched visual inspection:

* galaxy was clearly isolated in PCA and UMAP
* cat and dog overlapped more strongly
* car was moderately separated
* dog was reasonably coherent

### Main Takeaway

Galaxy images and captions form the most distinct multimodal concept cluster in this controlled dataset.

Cat is the hardest class, likely due to overlap with dog-like visual and semantic features.

## Key Scientific Findings

### Finding 1 — CLIP Produces Aligned Image and Text Embeddings

CLIP maps both image and text inputs into 512-dimensional embeddings.

```text
image embedding: (1, 512)
text embedding:  (1, 512)
```

This confirms that image and text can be compared directly using cosine similarity.

---

### Finding 2 — CLIP Performs Semantic Retrieval

Exact caption retrieval is too strict when multiple captions per class exist.

Many apparent retrieval failures are semantically valid.

Example:

```text
galaxy image → another galaxy caption
```

This means CLIP is retrieving based on semantic similarity rather than exact row pairing.

---

### Finding 3 — Matching Image-Text Pairs Are More Similar Than Non-Matching Pairs

For the 20-sample dataset:

```text
matching_mean     = 0.2712
non_matching_mean = 0.1856
alignment_gap     = 0.0856
```

The positive alignment gap indicates successful image-text alignment.

---

### Finding 4 — A Modality Gap Exists

Joint PCA shows that image and text embeddings occupy different regions of the low-dimensional space.

The largest variance direction is largely modality-related.

Interpretation:

```text
shared semantic structure + modality offset
```

This is a central result of the project.

---

### Finding 5 — Galaxy Is the Most Distinct Class

Galaxy achieved the highest class separation score:

```text
galaxy separation = 0.1260
```

This suggests that galaxy images and captions form a strong multimodal cluster.

---

### Finding 6 — Cat Is the Hardest Class

Cat achieved the lowest class separation score:

```text
cat separation = 0.0808
```

Cat/dog overlap is the strongest source of confusion in the current dataset.

---

### Finding 7 — Multiple Analyses Agree

The following analyses tell a consistent story:

* retrieval analysis
* similarity matrix analysis
* cross-modal alignment
* PCA
* UMAP
* joint PCA
* class separation

All indicate that CLIP has meaningful semantic structure, a measurable image-text alignment signal, and a visible modality gap.

## Lessons Learned

### 1. Infrastructure From Previous Repos Helps

Starting with:

* `cli.py`
* `paths.py`
* `config_loader.py`
* `logger.py`
* `reproducibility.py`

made development faster and cleaner.

This confirmed that the standard research-repo structure should be reused for familiar ML and representation-learning projects.

### 2. Exact Matching Can Be Misleading

Once there are multiple samples per class, exact pair retrieval becomes too strict.

Class-level retrieval is often more meaningful.

### 3. More Samples Improve Visualization

With 4 samples, UMAP was essentially meaningless.

With 20 samples, PCA and UMAP became interpretable.

This reinforces the idea that visual analysis needs enough samples to support meaningful geometry.

### 4. PCA and Metrics Should Be Used Together

PCA/UMAP plots are useful, but they should not be interpreted alone.

In this project, the strongest conclusions came from agreement between:

* retrieval
* similarity matrices
* alignment gap
* class separation
* PCA/UMAP

### 5. Modality Gap Is Real

Joint PCA clearly showed image and text embeddings separated along a modality axis.

This is an important idea for future multimodal alignment work.
